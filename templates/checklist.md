# Validation Gate Checklists

> 각 Phase 전환 시 검증하는 게이트. 통과하지 못하면 다음 단계로 진행하지 않습니다.

---

## Gate 1: PRD 완료 → Architecture 시작 전

- [ ] 문제 정의가 한 문장으로 요약 가능한가?
- [ ] 타겟 사용자와 JTBD가 명확한가?
- [ ] Must Have 기능의 수용 기준(AC)이 테스트 가능한가?
- [ ] 성공 지표(KPI)가 측정 가능한가?
- [ ] 스코프가 합의되었는가? (Won't Have 명시)
- [ ] 토론 패널 5명 전원 합의?
- [ ] CEO 결정 필요 사항이 해결되었는가?

**통과 조건**: 전체 항목 체크 또는 CEO가 "진행" 승인

---

## Gate 2: Architecture 완료 → Story 분해 전

- [ ] 기술 스택이 확정되었는가?
- [ ] 데이터 모델이 PRD 기능을 모두 커버하는가?
- [ ] API 설계가 완료되었는가?
- [ ] 보안 설계가 포함되었는가?
- [ ] 성능 목표가 설정되었는가?
- [ ] 테스트 전략이 수립되었는가?
- [ ] Brownfield인 경우 기존 시스템 영향도 분석 완료?
- [ ] ADR이 작성되었는가?

**통과 조건**: 전체 항목 체크

---

## Gate 3: Story 작성 → TDD 개발 전

- [ ] 모든 Must Have Story가 작성되었는가?
- [ ] 각 Story에 수용 기준(AC)이 있는가?
- [ ] 변경 파일 목록이 명시되었는가?
- [ ] TDD 계획 (Unit/Integration/E2E)이 있는가?
- [ ] 엣지 케이스가 식별되었는가?
- [ ] Story 간 의존성이 정리되었는가?
- [ ] 구현 순서가 결정되었는가?

**통과 조건**: Must Have Story 전체 작성 완료

---

## Gate 4: TDD 개발 → E2E 테스트 전

- [ ] 모든 Unit Test 통과?
- [ ] 모든 Integration Test 통과?
- [ ] 기존 테스트 깨지지 않음?
- [ ] 코드 컨벤션 준수?
- [ ] lint/typecheck 통과?
- [ ] 보안 취약점 없음?

**통과 조건**: 전체 테스트 Green

---

## Gate 5: E2E 테스트 → PR 생성 전

- [ ] 모든 Playwright E2E 시나리오 통과?
- [ ] 수용 기준(AC) 전체 충족?
- [ ] 성능 기준 충족?
- [ ] 접근성 기준 충족? (해당 시)

**통과 조건**: 전체 E2E Green + AC 충족

---

## Quick Gate (축약 모드)

CEO가 "토론 스킵" 또는 "빠르게"라고 한 경우:

- [ ] 핵심 기능이 정의되었는가?
- [ ] TDD 테스트 목록이 있는가?
- [ ] 최소한의 수용 기준이 있는가?

**통과 조건**: 3개 항목 체크
