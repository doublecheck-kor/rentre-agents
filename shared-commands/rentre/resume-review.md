## 역할
채용 이력서 평가 전체 파이프라인을 자동화합니다. 1차 OpenAI 스크리닝 → 사전 필터링 → Claude 2차 재평가 → Notion 인터뷰 페이지 생성 → 슬랙 알림.

## 핵심 정보
- Slack DM channel: {{SLACK_DM_CHANNEL}}
- 이력서 프로젝트 경로: {{RESUME_PROJECT_PATH}}
- 타임존: {{TIMEZONE}}

## 트리거
- 크론 자동 실행 (매일 평일 08:03)
- "이력서 평가해줘", "resume review", "flow 대로 진행해줘" 등

## 전체 플로우

```
Step 1: OpenAI 1차 스크리닝
Step 2: 사전 필터링 (외국인 탈락 + 재직 상태 확인)
Step 3: Claude 2차 재평가 (A 이상 후보자)
Step 4: Notion 인터뷰 페이지 생성 (A- 이상 후보자)
Step 5: 슬랙 결과 알림
```

## 지시

각 Step을 순서대로 실행한다. Step 간 결과를 다음 Step의 입력으로 전달한다.
신규 평가 대상이 없으면 사용자에게 알리고 종료한다.

## Step 2: 사전 필터링

1차 스크리닝 결과에서 신규 평가 대상을 추출한 후, 각 이력서 PDF를 읽어 아래를 확인한다.

### 2-1. 외국인 지원자 필터

이력서의 이름, 국적, 언어, 학력 등을 종합 판단:
- 한국어가 아닌 이력서 → 탈락
- 외국 국적 명시 → 탈락
- 외국 대학만 졸업 (한국 학력 없음) → 탈락

**탈락 시**: 등급 D, 사유 "외국인 지원자 (한국인만 채용)", Claude 재평가 스킵

### 2-2. 재직 상태 필터

이력서에서 최근 직장의 재직/퇴사 상태를 확인:
- **통과**: 현재 재직중 또는 퇴사 3개월 이내
- **감점**: 퇴사 3~6개월 → 한 등급 감점, 면접 시 공백 사유 확인 필요
- **탈락**: 퇴사 6개월 초과 → C 등급 상한, 사유 "경력 공백 과다"
- **불명확**: 면접 확인 포인트에 추가

## Step 3: Claude 2차 재평가

**대상**: OpenAI 1차에서 A 이상 + 사전 필터링 통과한 후보자

에이전트를 **병렬**로 실행하여 BE/FE 동시 평가한다.

### 필요 파일 (rentre-check-jd 프로젝트)
- JD: `jd/rentre_BE_jd.txt`, `jd/rentre_FE_jd.txt`
- JD Map: `jdmap/be_map.txt`, `jdmap/fe_map.txt`
- 이력서: `이력서/be/*.pdf`, `이력서/fe/*.pdf`

### 평가 항목
1. 후보자 기본 정보 (이름, 경력, 학력)
2. JD 자격요건 매칭표 (항목 | O/△/X | 근거)
3. JD 우대사항 매칭표 (항목 | O/△/X | 근거)
4. JD Map 연차별 기대수준 대조표 (영역 | 기대수준 | 충족여부)
5. 이력서 정리 수준 (문제→원인→개선→결과, 수치, 맥락, 경력 흐름)
6. 종합 평가 (등급 + 리스크 + 면접 확인 포인트)

### 엄격한 등급 기준
| 등급 | 기준 |
|------|------|
| A+ | JD 자격요건 100% + 우대사항 75% 이상 + 이력서 정리 최상 |
| A | 자격요건 90% 이상 + 이력서 정리 우수 |
| A- | 자격요건 85% 이상 + 이력서 정리 양호, 또는 매칭 우수하나 일부 영역 부족 |
| B+ | 매칭 있으나 이력서 정리 아쉬움, 또는 경력 미달이지만 역량 우수 |
| B | JD 일부 매칭 + 이력서 구성 불균형 |
| C | JD 미달 또는 이력서 정리 미흡 |
| D | 자격요건 크게 미달 |

### 감점 요인
- 구체적 수치 없는 성과 기술 → 감점
- 기술 키워드만 나열 → B+ 이하
- 이력서 정리 수준 B 이하 → 최대 B+ 제한

## Step 4: Notion 인터뷰 페이지 생성

**대상**: Claude 재평가 결과 **A- 이상** (A-, A, A+) 후보자만

### Notion DB 정보
- Data Source: `collection://18348a03-3208-807b-84fc-000bf7cad26d`
- 템플릿 ID: `30548a03-3208-8010-bd1a-d5cfe3622f8d`

### 페이지 생성 방법
```
mcp__claude_ai_Notion__notion-create-pages:
  parent: { data_source_id: "18348a03-3208-807b-84fc-000bf7cad26d" }
  pages: [{
    template_id: "30548a03-3208-8010-bd1a-d5cfe3622f8d",
    properties: {
      면접자: "{이름} 1차 실무(기술) 인터뷰",
      기술 스택: "Back-End" | "Front-End",
      경력: {연수},
      유형: '["직무인터뷰"]',
      상태: "시작 전",
      평가: "{등급}"
    }
  }]
```

### AI 문제 출제 (기술 압박 섹션에 추가)

기존 템플릿의 기술 압박 토글 마지막 줄 뒤에 `update_content`로 추가한다.

**이력서 기반 맞춤 설계 문제 3개:**
- 형식: 질문 → 상황 → 요구사항 → 슬라이드 포함 항목
- 이력서에 기술된 프로젝트/경험을 렌트리 상황에 맞게 변환

**기술 압박 질문 4개:**
- 이력서에서 주장한 역량의 깊이를 검증하는 압박 질문
- 구체적 구현 세부사항, 트레이드오프, 실패 경험을 묻는 질문

### Notion Markdown 주의사항
- 기술 압박 토글 내부이므로 탭 1개(\t) 들여쓰기
- 토글 내용은 탭 2개(\t\t)
- old_str: `5. API 일부 실패 시에도 화면 전체가 깨지지 않아야 한다.` + `<empty-block/>`
- new_str: 기존 줄 + `---` 구분선 + AI 생성 문제들
